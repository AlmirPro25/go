# Diretrizes de Trabalho com Projeto Existente

## REGRAS FUNDAMENTAIS

### 1. NÃO CRIAR FUNCIONALIDADES NOVAS
- **NUNCA** adicionar novas funcionalidades sem autorização explícita
- **SEMPRE** trabalhar apenas com o que já existe no código
- **PRIMEIRO** pesquisar e entender o que já está implementado
- **DEPOIS** ajudar com melhorias ou correções do código existente

### 2. PROCESSO OBRIGATÓRIO ANTES DE QUALQUER AÇÃO
1. **ANALISAR** o código existente completamente
2. **IDENTIFICAR** as funcionalidades já implementadas
3. **ENTENDER** como o sistema funciona atualmente
4. **TRABALHAR** apenas dentro do escopo existente

### 3. QUANDO POSSO SUGERIR ALGO NOVO
- **APENAS** se o usuário deixar EXPLICITAMENTE claro que quer algo novo
- **SEMPRE** perguntar para confirmar antes de implementar
- **NUNCA** assumir que o usuário quer funcionalidades adicionais

### 4. FOCO DO TRABALHO
- Melhorar código existente
- Corrigir bugs no que já existe
- Otimizar funcionalidades atuais
- Explicar como o sistema funciona
- Ajudar com configurações do que já está implementado

### 5. O QUE NÃO FAZER
- ❌ Criar novos componentes sem autorização
- ❌ Adicionar novas bibliotecas sem permissão
- ❌ Implementar novos recursos "por conta própria"
- ❌ Modificar a arquitetura sem solicitação explícita
- ❌ Criar sistemas de documentação não solicitados

### 6. SEMPRE LEMBRAR
- O projeto já tem suas funcionalidades definidas
- Meu papel é ajudar com o que JÁ EXISTE
- Qualquer mudança deve ser solicitada explicitamente
- Confirmar sempre antes de implementar algo novo
